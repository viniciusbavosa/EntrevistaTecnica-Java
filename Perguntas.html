Explique o que é uma API.

Uma API é um meio de comunicação entre um cliente e um servidor. Esse conceito é abrangente e pode ser interpretado da seguinte forma: Um cliente é algo que requisita informações e um servidor é algo que responde ao cliente com as informações requisitadas. Portanto, podem existir APIs que conectam aplicações na Web, como também podem existir APIs locais, que recebem um dado, o processam e devolvem esse dado a quem requisitou.

Quais são as principais diferenças entre um serviço e um programa na programação?

Segundo a Teoria da computação, um programa de computador (Turing Machine) possui o seguinte comportamento: Ele recebe um input, retorna um output e então é encerrado.

Já um serviço é um tipo especial de programa que nunca é encerrado. Ele se mantem em execução contínua aguardando por requisições, geralmente através de uma rede, para atender a solicitações enviadas por clientes (como programas ou usuários).

Exemplo de programa: Calculadora Exemplo de serviços: Servidores

Embora essa explicação seja teórica, na prática esses dois conceitos muitas vezes se sobrepõem. Por exemplo, atualmente, diversos programas podem continuar em execução em segundo plano, aguardando interações do usuário.

Como você implementaria autenticação em uma API?

Utilizaria JWT como um token de acesso. Ao logar na aplicação, as informações do usuário seriam enviadas para um endpoint específico da minha API, onde seriam processadas, validadas e sanitizadas. Se forem válidas, a API retornará um JWT access token como cookie para o cliente. Após esse processo, a cada nova requisição feita por esse cliente válidado, o servidor irá verificar se o JWT enviado com o cookie é válido e então permitir acesso as páginas protegidas da aplicação.

Explique o conceito de Concorrencia e Paralelismo

Concorrência é uma abordagem de execução de código bastante utilizada em linguagens single-threaded. Com ela, o sistema simula a execução de multíplas tarefas ao mesmo tempo, mas na realidade, o que está acontecendo é que o sistema operacional está utilizando um "agendador de tarefas". Quando o código é executado, o sistema operacional executa um pouquinho de cada tarefa e as alterna, simulando o paralelismo. Esse comportamento permite que código assíncrono seja executado e não pause a execução do seu programa.

Já o paralelismo se refere a real execução de código simultaneo. Isso porque o parelismo utiliza o conceito de multi-threaded, delegando as tarefas síncronas e assíncronas a diferentes threads e as executando ao mesmo tempo.

O que são estados na programação?

Estado refere-se à capacidade de uma aplicação de "memorizar" (i.e., armazenar) dados ao longo do tempo, permitindo que determinados dados sejam mantidos e reutilizados em diversas operações durante a execução. Ele representa a condição atual de um sistema, variável, objeto ou componente, influenciando seu comportamento e as operações seguintes.

O que é JSON?

O JSON é um formato de serialização de dados. Ele é útil para troca de dados na web e em diversos outros campos da programação. O JSON substituiu o antigo XML por sua legibilidade e simplicidade, tanto para humanos quanto para máquinas. O JSON transforma objetos e estruturas de dados em texto planos, onde então podem ser enviados através do protocolo HTTP, por exemplo, e então desserializado. Além da web, JSON é utilizado em diversas áreas da programação, incluindo armazenamento de configurações, comunicação entre serviços e integração de APIs.

O protocolo HTTP é um protocolo stateless ou statefull? Cite um exemplo de protocolo stateless e statefull

O protocolo HTTP é stateless. Isso significa que cada requisição enviada por um cliente é tratada de forma independente, sem que o servidor mantenha informações sobre requisições anteriores. Como resultado, toda a informação necessária para processar a requisição precisa ser enviada junto a ela, pois o servidor não armazena o estado da comunicação entre diferentes requisições. Isso permite que requisições vindas do mesmo cliente possam ser atendidas por qualquer instância do serviço, garantindo escalabilidade e distribuição de carga.

O protocolo HTTP foi originalmente projetado para a comunicação entre navegadores e servidores web, para isso ele precisava ser leve, de fácil escalabilidade e eficiente. Para alcançar esse objetivo, ele foi desenvolvido de forma stateless.

O protocolo SMTP é um exemplo de protocolo com estado. Esse protocolo coordena transmissão de dados entre emails, clientes e servidores. Nele, a requisição passa por uma sequência de etapas (HELO, MAIL FROM, RCPT TO, DATA, QUIT), e e cada etapa depende do sucesso da anterior, exigindo que o servidor mantenha o contexto da sessão até que o e0mail seja totalmente transmitido.

Você tem experiência em construir APIs RESTful? Explique o conceito da Arquitetura REST

Arquitetura REST é um conjunto de regras e princípios criado por Roy Fielding. Sua implementação padroniza a comunicação entre aplicações, tornando a troca de informação simples e efetiva.

Em um sistema RESTful, o servidor não mantém estado entre as requisições (i.e stateless), o que significa que cada solicitação deve incluir todas as informações necessárias para que o servidor possa processá-la de forma independente. Normalmente, a comunicação ocorre por meio de JSON, embora outros formatos também possam ser utilizados.

Além disso, a arquitetura REST estabelece uma clara separação entre o cliente, que consome os recursos, e o servidor, que os fornece. Essa divisão permite que cada parte evolua de forma independente, contribuindo para a escalabilidade e a manutenção do sistema. Por fim, sistemas RESTful devem expor uma API consistente baseada na semântica dos métodos HTTP, como GET, POST, PUT e DELETE, concentrando-se na manipulação de recursos em vez de ações específicas, o que garante previsibilidade e consistência nas operações.

O que é um load balancer?

Load balancer é um tipo de proxy que pode se conectar a vários servidores. O seu trabalho consiste em controlar e distribuir o fluxo de requisições entre esses servidores para que não fiquem sobrecarregados.

O que é idempotência? E como ela se relaciona a APIs RESTful?

A idempotência refere-se à característica de uma operação em que a execução repetida da mesma ação, com os mesmos parâmetros, terá sempre o mesmo efeito, independentemente de quantas vezes a ação seja realizada.

Como uma API RESTFul segue o protocolo HTTP, então ela também deve ser idempotênte. Isso significa que a API deve receber uma mesma requisição varias vezes e sempre retornar a mesma resposta, com exceção de requisições com método POST que não são idempotentes,

Eu gosto de associar idempotência com o conceito de funções puras da programação funcional. Dado um input, o output deve ser o mesmo sempre.

Java

Explique como o Java gerencia a memória e quais são as principais diferenças em relação a linguagens como C.

O Java gerencia a memória através do coletor de lixo ou Garbage Collector(GC), que remove objetos não referenciados. Isso permite que o programador não precise se preocupar com o gerenciamento de memória.

Em linguagens como C, o gerenciamento de memória é feito manualmente pelo desenvolvedor, alocando memória (malloc, calloc, realloc) e liberar (free) manualmente a memória, aumentando a chance de vazamentos e acessos inválidos.

Tanto Java, quanto C utilizam Stack e Heap para gerenciar memória, porém com algumas diferenças:

  • Java
    • Variáveis primitivas e referências a objetos são armazenadas na Stack
    • Objetos são armazenados no heap, sendo liberados pelo GC quando não há mais referências a eles
  • C
    • Variáveis locais e ponteiros são armazenas na Stack
    • Dados alocados dinamicamente (malloc) ficam no heap, precisando ser liberados manualmente

Java é uma linguagem fortemente influenciada pelo paradigma de Programação Orientada a Objetos. Fale um pouco sobre os conceitos de POO.

1. Classes e Objetos

  • Classes: São moldes ou “blueprints” que definem atributos (dados) e métodos (comportamentos) comuns a um conjunto de objetos.
  • Objetos: São instâncias das classes, representando entidades do mundo real com estados (valores dos atributos) e comportamentos (métodos).

2. Encapsulamento

Oculta os detalhes internos de uma classe, expondo apenas o necessário. Em Java, isso é feito utilizando modificadores de acesso como private, protected e public para proteger os dados e garantir um acesso controlado.

3. Herança

Permite que uma classe “filha” herde atributos e métodos de uma classe “mãe”. Isso promove o reuso de código e facilita a criação de hierarquias, onde classes mais especializadas podem estender os comportamentos de classes mais gerais.

4. Polimorfismo

Refere-se à capacidade de uma mesma operação se comportar de maneira diferente em classes distintas. Em Java, isso é frequentemente implementado através da sobrescrição (override) de métodos, possibilitando que uma chamada de método seja resolvida dinamicamente conforme o objeto que a recebe.

5. Abstração

Consiste em representar os aspectos essenciais de um objeto, ignorando detalhes não relevantes. Em Java, classes abstratas e interfaces são usadas para definir contratos que especificam o que um objeto deve fazer, sem se preocupar com a implementação exata.

Esses conceitos colaboram para a criação de sistemas modulares, reutilizáveis e de fácil manutenção, permitindo que os desenvolvedores mapeiem problemas do mundo real para soluções de software de forma mais eficaz.

Quais são os modificadores de acesso em Java e suas funções?

  • Default (sem nenhum modificador): O acesso é permitido apenas dentro do mesmo pacote
  • Private: O mais restritivo, membros privados são acessíveis somente dentro da classe onde foram declarados, nem mesmo classes do mesmo pacote podem acessá-los.
  • Protected: Podem ser acessados dentro do mesmo pacote ou subclasses de pacotes diferentes.
  • Public: Podem ser acessados de qualquer lugar do código.

O que são interfaces em Java e como elas se diferenciam de Classes?

Em Java, uma interface é um tipo especial de estrutura que define um conjunto de métodos que uma classe deve implementar, sem fornecer nenhuma implementação real desses métodos (até o Java 8).

Enquanto classes podem conter atributos e métodos com implementação, interfaces servem apenas como um contrato que outras classes precisam seguir.

O que o princípio SOLID representa?

O SOLID representa os 5 princípios de Java que são:

  • S: Princípio de responsabilidade Única. Uma classe deve ter uma única responsabilidade.
  • O: Princípio do Aberto/Fechado. Um módulo deve estar aberto para extensão, mas fechado para modificação
  • L: Princípio da Substituição de Liskov. Subtipos/subclasses devem ser substituíveis por seus tipos base sem alterar o comportamento correto do programa. Por exemplo, temos uma classe Retangulo, uma classe Quadrado que a herda não deve quebrar o comportamento esperado.
  • I: Princípio da Segregação de Interfaces. Uma classe não deve ser forçada a implementar métodos que não utiliza.
  • D: Princípio da Inversão de Dependência. Módulos de alto nível não devem depender de módulos de baixo nível. Ambos devem depender de abstrações.

Quais são os tipos de dados primitivos

Em Java, os tipos primitivos são:

  • Byte: O tipo de dado byte é um inteiro que corresponde a 1 Byte ou 8 bits na memória do computador. Ele pode armazenar números em um intervalo de -128 a 127, pois o Java o armazena com sinal. Esse tipo de dado é útil para economizar memóri em matrizes grandes, onde a economia de memória é realmente necessária. Eles também podem ser usados no lugar de int pois seus limites (-128 a 127) ajudam a tornar o código mais claro. Isso significa que, se você tiver uma variável que nunca precisará armazenar valores além desse intervalo, usar byte em vez de int pode deixar mais evidente para outros desenvolvedores que aquele valor tem um limite específico.
  • Short: O tipo de dado short é um inteiro que corresponde a 2 Bytes ou 16 bits com sinal na memória do computador. Ele pode armazenar números em um intervalo de -32,768 a 32,767. Assim como em byte, você pode utilizar esse tipo de dado para economizar memória em matrizes, além de servir como documentação.
  • Int: Por padrão, o tipo de dado int é um inteiro que corresponde a 4 Bytes ou 32 bits com sinal na memória do computador. Ele pode armazenar números em um intervalo de -231 a 231-1, ou de −2.147.483.648 a 2.147.483.647.
  • Long: O long é um inteiro que corresponde a 8 bytes ou 64 bits na memória do computador. Ele pode armazenar números em um intervalo de -263 a 263-1, ou de -9.223.372.036.854.775.808 a 9.223.372.036.854.775.807.
  • Float: Tipos de dado float são usados para representar números de ponto flutuante, ou seja, números que possuem casas decimais. Ele corresponde a 4 bytes ou 32 bits na memória do computador. Ele pode representar números em um intervalo de - 3.402823 x 1038 a 3.402823 x 1038. O float possui uma precisão limitada a 7 dígitos decimais. Se for necessário uma maior precisão, utilize o tipo double. Nunca utilize esse tipo de dado para valores precisos, como valores monetários.
  • Double: O tipo double é o padrão para representar números de ponto flutuante. Ele oferece maior precisão e maior intervalo de valores do que float. Um dado do tipo double ocupa 8 bytes ou 64 bits na memória e oferece uma precisão de 15 a 16 dígitos significativos. Assim como em float, não utilize esse tipo para valores monetários.
  • Boolean O tipo boolean representa apenas dois valores: true e false. Segundo a documentação do Java, esse tipo representa 1 bit na memória, porém seu "tamanho" não é definido com precisão.
  • Char: Um dado do tipo char representa um único caractere Unicode de 16 bits, o que significa que ele pode representar praticamente qualquer símbolo ou caractere de qualquer língua, como letras, números e outros caracteres especiais. Esse tipo ocupa 2 bytes ou 16 bits na memória. Seu intervalo de valores é de 0 a 65,535 (ou de \u0000 a \uffff em notação Unicode), representando os primeiros 65.536 caracteres do conjunto Unicode.

Me fale um pouco sobre a ordem de execução dos blocos de inicialização e construtores em Java.

Em Java, a ordem de execução dos blocos de inicialização e dos contrutores segue uma sequência específica. A ordem de execução é importante para entender como o código é inicializado em uma classe e seus objetos.

    Ordem de Execução

  • Bloco de inicialização estático (Static Block) é executado uma única vez quando a classe é carregada pela primeira vez na memória pela JVM, ou seja, antes da criação de qualquer instância da classe. Blocos de inicialização são executados na mesma ordem em que aparecem no programa, mas antes do construtor da classe.
  • Blocos de inicialização de instâncias são executados sempre que uma nova instância da classe é criada. Ele é executado antes do construtor e após a criação do objeto (mas antes da execução do corpo do construtor). Esse bloco é útil para inicializações comuns que precisam ser feitas em todos os contrutores da classe.
  • Construtor da classe é executado após o bloco de inicialização de instância e é responsável por inicializar o objeto quando ele é criado.

Qual a diferença entre funções Lambda e Closures?

Funções lambda foram introduzidas no Java 8 para simplificar o processo de criação de funções. Com elas, podemos declarar funções anonimas, ou seja, sem um nome, tipo de retorno e modificador de acesso definido.

Uma closure é uma função que "lembra" e mantem acesso ao escopo léxico onde foi declarada.

Quais novas funções foram adicionadas no Java 8?

  • Lambda Expressions − a new language feature allowing us to treat actions as objects
  • Method References − enable us to define Lambda Expressions by referring to methods directly using their names
  • Optional − special wrapper class used for expressing optionality
  • Functional Interface – an interface with maximum one abstract method; implementation can be provided using a Lambda Expression
  • Default methods − give us the ability to add full implementations in interfaces besides abstract methods
  • Nashorn, JavaScript Engine − Java-based engine for executing and evaluating JavaScript code
  • Stream API − a special iterator class that allows us to process collections of objects in a functional manner
  • Date API − an improved, immutable JodaTime-inspired Date API

Spring Boot

O que é Spring Boot?

Spring Boot é um framework open-source que facilita o desenvolvimento de aplicações Java baseadas no Spring Framework. Ele foi projetado para simplificar a configuração e a inicialização de projetos, permitindo que os desenvolvedores criem aplicativos Java rapidamente, com o mínimo de configuração manual.

Principais características do Spring Boot:

  • Configuração Automática: O Spring Boot fornece uma série de configurações automáticas que ajudam a reduzir a quantidade de configuração necessária. Com isso, o desenvolvedor pode se concentrar mais na lógica de negócios e menos em detalhes de configuração.
  • Aplicações Standalone: O Spring Boot permite criar aplicativos Java autossuficientes (standalone), sem a necessidade de um servidor de aplicação externo como o Tomcat ou o Jetty, pois ele já embute servidores de aplicação dentro do próprio projeto.
  • Dependências Gerenciadas: O Spring Boot inclui uma abordagem de dependências "starter", ou seja, pacotes de bibliotecas pré-configurados que ajudam a adicionar rapidamente funcionalidades ao projeto, como banco de dados, segurança e web services.
  • Rápida Inicialização: O Spring Boot é projetado para permitir o desenvolvimento rápido de aplicativos, com uma estrutura simplificada, o que é ideal para protótipos, MVPs e até mesmo soluções em produção.
  • Atendimento a Microserviços: Ele se adapta bem à arquitetura de microserviços, tornando o desenvolvimento e a implantação de sistemas distribuídos mais fácil.
  • Actuators: O Spring Boot inclui módulos de gerenciamento e monitoramento, chamados de "actuators", que fornecem métricas, informações sobre o estado da aplicação, logs, e muito mais.

Com o Spring Boot, é possível criar desde simples APIs RESTful até sistemas mais complexos e escaláveis com segurança, persistência e integração com outros sistemas.

Qual a finalidade do Spring Boot e como ele facilita o desenvolvimento de aplicações Java?

O Spring Boot foi criado para simplificar e agilizar o desenvolvimento de aplicações Java baseadas no Spring Framework. Ele elimina a necessidade de configurações manuais complexas, oferecendo autoconfiguração e dependências agrupadas (starters) que permitem iniciar projetos rapidamente. Além disso, o Spring Boot permite a criação de aplicações standalone com servidores embutidos, o que facilita o processo de deploy e torna o desenvolvimento mais produtivo.

Como o Spring Boot lida com diferentes perfis de ambiente (development, test, production) e como você os configura?

O Spring Boot permite a criação de configurações específicas para diferentes ambientes por meio do conceito de perfis. Por exemplo, é possível definir arquivos como application-dev.properties e application-prod.properties, cada um com configurações apropriadas para o ambiente de desenvolvimento ou produção, respectivamente. O perfil ativo é especificado pela propriedade spring.profiles.active, possibilitando que a aplicação carregue automaticamente as configurações corretas conforme o ambiente em que está sendo executada.

Como você integraria o Spring Security a uma aplicação Spring Boot para gerenciar autenticação e autorização?

A integração do Spring Security em uma aplicação Spring Boot é feita adicionando a dependência spring-boot-starter-security, que traz uma série de configurações e filtros de segurança prontos para uso. A partir daí, é possível personalizar regras de autenticação e autorização por meio de classes de configuração anotadas com @EnableWebSecurity, definindo quais endpoints devem ser protegidos e como os usuários serão autenticados (por exemplo, através de um UserDetailsService). Isso garante que somente usuários autorizados possam acessar determinados recursos da aplicação.

Como você gerencia logs em uma aplicação Spring Boot e quais práticas recomendadas utiliza para configurá-los?

O Spring Boot utiliza, por padrão, o Logback como framework de logging, mas também suporta outros frameworks como Log4j2. A configuração de logs pode ser feita por meio do arquivo application.properties ou de um arquivo de configuração específico, como logback.xml. Entre as práticas recomendadas estão definir níveis de log apropriados para diferentes pacotes, configurar logs para diferentes ambientes (por exemplo, maior detalhamento em desenvolvimento e menos verbosidade em produção) e, quando necessário, integrar a aplicação com sistemas de gerenciamento centralizado de logs para facilitar o monitoramento e a análise de problemas.

Qual a abordagem do Spring Boot para trabalhar com persistência de dados, por exemplo, usando Spring Data JPA e Hibernate?

O Spring Boot simplifica o acesso e a manipulação de dados ao integrar o Spring Data JPA e o Hibernate de forma automática. Ao incluir a dependência spring-boot-starter-data-jpa, o framework configura automaticamente o EntityManager, o datasource e outras dependências necessárias. Os desenvolvedores podem definir entidades com anotações como @Entity e utilizar interfaces que estendem JpaRepository para realizar operações CRUD sem precisar implementar manualmente cada método, o que acelera o desenvolvimento e mantém o código mais limpo.

De que forma o Spring Boot implementa a injeção de dependências e quais são as principais anotações utilizadas, como @Autowired, @Component, @Service e @Repository?

O Spring Boot utiliza o contêiner de Inversão de Controle (IoC) do Spring para implementar a injeção de dependências. Com isso, classes anotadas com @Component, @Service, @Repository ou @Controller são automaticamente registradas como beans. A anotação @Autowired permite que o Spring resolva e injete essas dependências automaticamente em outras classes, promovendo um acoplamento baixo e facilitando a manutenção e os testes da aplicação.

O que é o framework Spring? e o Spring Boot?

O framework Spring é uma plataforma open-source para o desenvolvimento de aplicações em Java, projetada para facilitar a criação de sistemas corporativos robustos, escaláveis e de fácil manutenção. Ele promove conceitos fundamentais como a Inversão de Controle (IoC) e a Injeção de Dependências, que ajudam a desacoplar os componentes do sistema, tornando o código mais modular e testável. Além disso, o Spring oferece uma série de módulos que atendem a diferentes necessidades, como desenvolvimento web (Spring MVC), persistência de dados (Spring Data), segurança (Spring Security) e muito mais, permitindo a integração com diversas tecnologias e frameworks para a construção de soluções completas.
